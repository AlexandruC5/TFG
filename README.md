## Resumen
En el desarrollo de videojuegos el diseño del mapa es una parte clave y cada vez la audiencia tiene una demanda más alta de calidad tanto en el diseño como en los materiales que se visualizan dentro de las escenas. Los diseñadores, requieren de ciertos elementos para llenar el mapa, estos elementos suelen ser objetos 3D que presentan una decoración o algún elemento repetitivo como pueden ser los árboles, carreteras o diferentes objetos cotidianos o abstractos, dependiendo del encuadre artístico que tenga el videojuego. Estos objetos 3D son modificados ampliamente y se usan varias iteraciones para decidir su posición, si la estética es correcta y si encuadra con el arte del juego. Este proceso, consume muchos recursos porque se necesita un artista para la creación de dichos assets y luego se requiere del diseñador para que se coloque y se itere el nivel con dichos assets. 

Por es razón, este proyecto se enfoca en crear una herramienta para facilitar la generación de modelos de objetos 3D simples para llenar una escena y dar un abanico de posibilidades más amplio a un diseñador consumiendo menos recursos y asegurando una calidad mayor ya que se podrá realizar más iteraciones con distintos objetos 3D. 

Para crear dicha herramienta se ha investigado cómo generar modelos 3D de manera procedural y programática y como validar que los elementos generados cumplen con los requisitos que pide el usuario.

Para la generación de los modelos se han investigado diferentes métodos usados en la historia para generar elementos 3D de manera procedural y los algoritmos usados. Seguidamente, para la validación de los objetos creados se ha investigado cómo crear un modelo basado en una red neuronal para identificar si los objetos creados son correctos y concuerdan con la petición del usuario. 

Finalmente, se han cumplido la gran mayoría de objetivos pero no en su totalidad, en el proyecto se pueden ver el progreso de cómo se han ido cumpliendo dichos objetivos y el por qué no se han podido cumplir los restantes.
